## 2. 技術構成（シンプルな静的サイト・ビルド不要）

| ファイル | 役割 |
|---|---|
| `index.html` | 画面とCSS（全部インライン）。**ホーム/単語クイズ/並べ替え/ずかん**の4画面＋下部ナビ |
| `app.js` | アプリのロジック全部 |
| `data.js` | 問題データ（`UNITS`）＋旅データ（`JOURNEY`）＋名物（`PREF_GOODS`）。**ここを編集すれば追加できる** |
| `japan-map.js` | **本物の日本地図**データ（`JAPAN_PATH`/`PREF_XY`/`OKINAWA_PATH`等）。出典:地球地図日本(GSI)。自動生成物（下記） |
| `manifest.json` | PWA設定（名前「れいなの英語ドリル」、orientation: any） |
| `sw.js` | サービスワーカー（オフライン＋自動更新）。`CACHE` 番号を上げると更新が確実に伝わる |
| `icon.svg` | アプリアイコン（🐣＋ABC） |

- 保存は端末内 `localStorage`：`eigo-drill-v1`（セーブ本体）と `eigo-drill-face`（顔写真。任意・非公開）。サーバー・ログイン・DBなし。
- フレームワーク無し（素のHTML/CSS/JS）。

### 日本地図データ（japan-map.js）の作り直し方
`japan-map.js` は外部GeoJSONから生成した成果物（リポジトリにコミット済み）。作り直す場合：
1. 出典データを取得（dataofjapan/land、出典＝地球地図日本/GSI、非営利は出典明記でOK）:
   `japan.geojson`（約13MB）と `prefecturalCapital.csv` を
   `https://raw.githubusercontent.com/dataofjapan/land/master/...` から curl。
2. 生成スクリプト `generate-map.js`（※前回はscratchpadに置いた＝リポジトリには無い）を node 実行。
   海岸線をDouglas-Peuckerで簡略化、沖縄を本土と分離出力、県マーカーは県庁所在地の実緯度経度→投影。
3. 出力 `japan-map.js`（約26KB）をリポジトリにコピー。`index.html` は `data.js`→`japan-map.js`→`app.js` の順で読み込む。

## 3. 現在の機能

> 2026-06 大改修：旧「育成ペット＋着せ替えショップ」を廃止し、**「自転車で日本一周」**＋**ご当地名物図鑑**＋**忘却曲線SRS復習**に作り替えた。

- **単語4択クイズ**：日→英 / 英→日をランダム出題。選択肢は重複しないようdedupe済み
- **並べ替え問題**：日本語を見て単語タイルをタップで語順に。「難しい」チェックでダミー語ON
- **自転車で日本一周**（メインのごほうび）：正解すると距離が進む（**1問正解＝2km**, `JOURNEY.kmPerCorrect`）。三重県亀山を起点に**全47都道府県を一筆書きで巡り**亀山に帰る（合計 約4935km＝半年ペース, `segKm=105`）。県に**到着すると紹介カード**（ごほうび知識）＋コイン+10。全周で「日本一周たっせい」演出。
- **本物の日本地図**（`japan-map.js`＝出典:地球地図日本/国土地理院。`generate-map.js`で生成）：海岸線SVGパス＋県庁所在地の実座標(`PREF_XY`)でマーカー配置。**沖縄はインセット枠**（`OKI_BOX`/`okiToMain`）。`buildMapSVG()`で地図＋🚲＋ルート描画。
- **🏅○県 制覇！バッジ**（`#conquerBadge`、47で「全制覇！」演出）。
- **顔写真マーカー**：「顔写真をえらぶ」で端末内(`localStorage: eigo-drill-face`)に保存し🚲に合成。※公開リポジトリなので写真は端末内のみ・コミットしない方針。
- **ご当地名物図鑑**（「ずかん」タブ, `renderZukan()`）：47県×2〜4個＝**計143個**の名物（`data.js` の `PREF_GOODS`）。**到着済みの県**の名物を**コインで1個ずつ購入**（自分で選ぶ）→ 説明文つきで図鑑に集まる。県名・名物名・説明文で**検索**可。
- **忘却曲線SRS復習**：間違えた問題を `S.srs{box,due}` で管理。間違い→box0今日復習、正解で間隔 1日→3日→7日と延び、box4で**卒業（マスター）**。`startReview()`＝**きょうの復習**（期限ぶん）、`startWeak()`＝**にがて練習**（苦手全部）。到着モーダルの「道の駅で復習」もここへ。
- **コイン**：単語正解 +1、並べ替え正解 +2、県到着 +10
- **ストリーク（連続日数）・達成度バー（単元別）**
- **読み上げ（TTS）**：英語を発音（ブラウザ標準 speechSynthesis）
- **正解演出**：褒め言葉ポップ＋紙吹雪＋効果音（WebAudio）
- **PWA自動更新**：新バージョンを検知したら自動リロード（`controllerchange` ＋ sw の `skipWaiting`）

## 4. コンテンツ（data.js）

光村「Here We Go!」の単元構成に寄せた中1標準の**オリジナル問題**（教科書と完全一致ではない / 著作権配慮）。

- **全16単元・216単語・144文**
- Unit 1 be動詞 / 2 一般動詞 / 3 複数形と数 / 4 疑問詞 / 5 can / 6 三単現 / 7 現在進行形 / 8 過去形 / 9 命令文・Let's / 10 前置詞・There is / 11 曜日・月・季節 / 12 数と年齢 / 13 家族と人 / 14 食べ物・飲み物 / 15 形容詞 / 16 学校・身の回り
- データ形式（`data.js` 冒頭コメント参照）:
  - `words: [{ en:"英単語", ja:"日本語" }]`
  - `sentences: [{ ja:"日本語文", tokens:["正しい","語順","の","単語"] }]`（ピリオド/？は自動。tokensは単語のみ、先頭大文字）
- 検証ルール：全並べ替え文は「難しいモードでも正解タイルが必ず揃う」、全単語は「4択で正解肢が存在＋4肢ユニーク」を満たす（追加時は同様に確認すること）

## 6. これまでの設計判断（grill済み）

- 娘専用・固定データ・端末内保存と割り切ってシンプルに（サーバー/AI生成は使わない）
- 教科書は光村だが、正確な単語/例文は持っていないためオリジナルで網羅
- UIは漢字中心（中1が読める範囲。ペット名や「あなた」等はかな）
- ホームに飾りタイトル「🌸 れいなの英語ドリル 🐣」（グラデ文字＋揺れる絵文字）。地図カードの見出しは「🚲 自転車で日本一周にチャレンジしよう！」
- **れいなは横型タブレットで使用**。横型対応：`@media (orientation:landscape) and (min-width:740px)`。app最大幅**820px**／ホームは2カラム（地図＋練習）／**クイズは680px中央寄せ**／**ずかんは中央寄せの1カラム**（名物リストが入るため）。間延び防止で各カードを中央寄せにしてある。
- 顔写真は**公開リポジトリに載せない**方針（端末内localStorageのみ）。お子さんの顔写真を公開サイトに置かない判断。

## 7. セーブデータ構造（localStorage: eigo-drill-v1）

```
{ ver:4, exp, coins, level, streak, lastPlay,
  km,                 // 旅で進んだ距離
  visited:{"三重":true,...},  // 到着した県
  reached,            // 到達した spots のインデックス
  goods:{"三重#0":true,...},  // 買ったご当地名物
  correctWords:{}, correctSents:{},  // 達成度バー用
  srs:{ "word:u1:I":{box,due}, "sent:u2:3":{box,due} } }  // 忘却曲線の復習カード
```
- `load()` で自動移行：**ver<3** → 旅システム化（距離0スタート・コイン引継・三重を到着済みに・旧 maxStage/acc/owned 削除）。**ver<4** → 旧 `wrongPool` を `srs`（box0=今日復習）へ変換し `wrongPool` 削除。
- `due` は `dayNum()`（=`Math.floor(Date.now()/86400000)`、エポックからの日数）。SRS間隔 `SRS_INTERVALS={1:1,2:3,3:7}`、`SRS_GRADUATE=4`。
